# Roadmap

## Near term

- define the first concrete scenario templates
- formalize actor and state representations
- build a minimal local runtime
- document assumptions and logging conventions

## Medium term

- add modular payoff systems
- expand scenario libraries
- establish a calibration workflow
- compare isolated modules against integrated runs

## Longer term

- maintain an open hypothesis repository
- support repeatable benchmark scenarios
- improve reporting and visualization

## Current status

**March 2026:** early prototype with emphasis on concept clarification, architecture discipline, and validation design.
